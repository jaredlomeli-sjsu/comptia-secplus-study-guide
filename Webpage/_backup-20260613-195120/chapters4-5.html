Chapter 4 — Securing Your Network

Detection, wireless, and remote access

IDS vs IPS

Detect vs prevent — placement and method matter
AxisOption AOption B
ResponseIDS — detects and alerts (out-of-band, passive; sees a copy of traffic)IPS — sits inline and blocks in real time (preventive/corrective)
ScopeHIDS — agent on one host; sees encrypted traffic after decryption, local files/logsNIDS — sensors watching network segments; broad view, can't see inside encrypted payloads or host activity
MethodSignature-based — matches known attack patterns; low false positives but blind to zero-days (vulnerabilities with no patch/signature yet); needs constant updatesAnomaly/trend-based — builds a baseline of normal, flags deviations; can catch novel attacks but higher false positives; re-baseline after network changes

Getting traffic to a NIDS: port mirroring (SPAN) — switch copies traffic to a monitor port — or a hardware network tap. An aggregator combines multiple links into one feed for the sensor.

Vocabulary: false positive = alert with no attack (noise); false negative = attack with no alert (the dangerous one). An APT (Advanced Persistent Threat) is a well-funded, long-dwell adversary — typically nation-state — that anomaly detection and threat hunting target.

Wireless Fundamentals

Bands, surveys, and antennas
  • SSID — the network name an AP broadcasts. Disabling the broadcast hides it from casual users only — tools see it instantly (security by obscurity, not a real control).
  • Bands: 2.4 GHz = longer range, more interference (microwaves, Bluetooth), 3 non-overlapping channels (1/6/11). 5 GHz = faster, shorter range, many channels. 6 GHz = Wi-Fi 6E.
  • 802.11 standards: b/g (2.4 GHz) → n (both bands, "Wi-Fi 4") → ac (5 GHz, "Wi-Fi 5") → ax (Wi-Fi 6/6E).
  • Site survey — walk the space with a Wi-Fi analyzer to map coverage and interference; output is a heat map. Also used defensively to find rogue APs (wireless footprinting = mapping the wireless environment).
  • Antenna choice as a control: omnidirectional radiates in all directions; directional focuses the signal — aim coverage into the building, reduce AP power so the signal doesn't bleed into the parking lot.

Wireless Security Evolution — WEP → WPA3

Know what broke and what fixed it
GenerationCryptoVerdict
WEPRC4 with tiny 24-bit IVs that repeatbroken — crackable in minutes via IV attacks; never use
WPATKIP (still RC4) — stopgap for WEP-era hardwaredeprecated
WPA2 (802.11i)CCMP / AESStill common; PSK mode vulnerable to handshake capture + offline dictionary attacks on weak passphrases
WPA3AES; SAE (Simultaneous Authentication of Equals — Dragonfly handshake) replaces the PSK exchangecurrent — SAE resists offline dictionary attacks and adds forward secrecy

PSK / Personal mode

One shared passphrase for everyone. Fine at home; in business, no individual accountability and one leaver means re-keying every device.

Enterprise mode

802.1X + RADIUS: each user authenticates with their own credentials/certificate. Individual accountability, instant revocation. The AP and RADIUS server share a shared secret.

Open / OWE

Open = no encryption at all. WPA3's Enhanced Open (OWE) encrypts traffic on open networks without a password — fixes coffee-shop sniffing. A captive portal adds a web-based acceptance/payment page on top of open networks (free or paid tiers).

EAP Family & 802.1X

Certificate count is how you tell them apart

IEEE 802.1X = port-based network access control: nothing passes a switch port or AP until the supplicant (client) authenticates through the authenticator (switch/AP) to the authentication server (RADIUS). EAP is the framework carrying that authentication; the variants differ by who presents a certificate:

VariantServer CertClient CertNotes
PEAP✔✘Protects EAP inside a TLS tunnel; client uses password/MSCHAPv2
EAP-TTLS✔✘Like PEAP; supports legacy inner auth methods
EAP-FASToptional (PAC)✘Cisco's fix replacing LEAP; uses Protected Access Credential
EAP-TLS✔✔Most secure — mutual certificate authentication; heaviest to deploy (PKI for every client)

After authentication, client and AP derive the PTK (Pairwise Transient Key) — the per-session encryption key. RADIUS federation extends 802.1X across organizations (e.g., eduroam lets SJSU credentials work at other universities).

Exam cue: "requires certificates on both server and clients" = EAP-TLS. "Certificate only on the server" = PEAP or EAP-TTLS.

VPN Architectures

Tunnel scope and IPsec modes decide the answer
TypeDescription
Remote-access VPNIndividual user → corporate network, terminating at a VPN concentrator (dedicated device handling many encrypted tunnels)
Site-to-site VPNGateway-to-gateway tunnel permanently connecting two offices; transparent to users
Always-on VPNConnects automatically whenever the device is on an untrusted network — removes user error
Full tunnelAll traffic goes through the corporate tunnel — full inspection, more bandwidth load
Split tunnelOnly corporate-bound traffic uses the tunnel; general browsing goes direct — efficient, but that traffic bypasses corporate security controls
Clientless (HTML5 portal)Browser-based TLS access to specific apps — no VPN client install needed

Protocols & modes

  • IPsec tunnel mode — encrypts the entire packet (including the original IP header); used for traffic crossing the internet (site-to-site). Transport mode — encrypts payload only; host-to-host within a private network.
  • ESP (IP protocol 50) = encryption + auth; AH (IP protocol 51) = auth/integrity only, no encryption; IKE UDP 500 negotiates the keys.
  • L2TP UDP 1701 — tunneling with no encryption of its own → always paired as L2TP/IPsec.
  • SSTP TCP 443 — TLS-based VPN; rides HTTPS so it traverses restrictive firewalls.
Scenario cue: "remote workers' personal browsing is consuming corporate bandwidth" → switch to split tunnel. "Security needs to inspect all remote traffic" → full tunnel.

NAC — Network Access Control

Health-check the device before it joins

NAC inspects a connecting device's posture — patched? AV current? firewall on? — and grants full access, quarantines to a remediation VLAN, or blocks. Applies to wired, wireless, and VPN connections.

Agent-based

Software installed on the device gives deep, continuous inspection. A permanent agent persists; a dissolvable agent runs once for the check and removes itself (good for guest/BYOD).

Agentless

No install — scans/queries the device externally. Less depth, but works for devices you can't manage (guests, IoT).

Chapter 5 — Securing Hosts & Data

From hypervisors to handsets

Virtualization

Type I vs II, and the VM-specific risks
  • Hypervisor creates/runs VMs. Type I (bare-metal) runs directly on hardware (ESXi, Hyper-V server) — production. Type II (hosted) runs on top of a host OS (VirtualBox, VMware Workstation) — desktops/labs.
  • Host = physical machine; guest = each VM. Snapshot = point-in-time state you can revert to (take one before patching/testing). Replication copies VMs elsewhere for recovery.
  • VDI — desktops run centrally in the data center; users connect from thin clients (minimal devices). Data never lives on the endpoint.
  • Containers — app-level isolation sharing one OS kernel (Docker). Lighter/faster than VMs, but weaker isolation: a kernel compromise touches every container.
VM RiskDefinition / Mitigation
VM escapeAttacker breaks out of a guest into the hypervisor/host — the most severe VM attack. Mitigation: patch the hypervisor promptly
VM sprawlUnmanaged, forgotten VMs accumulating unpatched → inventory + lifecycle policies
Resource reuseMemory/storage reallocated between VMs without sanitization can leak the previous tenant's data

Shared responsibility: in every model, your data and your identities remain your responsibility. The exam loves "who patches the OS in IaaS?" — you do.

Hardening & Boot Integrity

Shrink the attack surface, then verify the boot chain
  • Hardening — disable unneeded services/ports/accounts, change defaults, apply secure configuration baselines.
  • Master image (gold image) — pre-hardened, pre-configured standard image deployed to all machines: consistency + faster, safer provisioning.
  • Configuration management enforces baselines and detects drift; change management ensures modifications are reviewed/approved/documented — preventing outages from ad-hoc changes.
  • Patch management — test patches first, deploy on schedule, verify installation; the single most effective vulnerability mitigation.

Firmware & the boot chain

TermMeaning
BIOS → UEFILegacy firmware vs its modern replacement; UEFI adds Secure Boot support. Updating firmware = flashing
Secure BootUEFI verifies the digital signature of bootloader/OS at each step — refuses to run unsigned/tampered code (stops rootkits/bootkits)
Measured BootEach boot stage is measured (hashed) into the TPM; doesn't block boot, produces evidence
Boot attestationThose TPM measurements are reported to a remote server which decides if the device is healthy enough to join the network
Distinction: Secure Boot blocks tampered code; Measured Boot records and reports it. Attestation = a third party verifying the measurements.

TPM, HSM, Disk Encryption & DLP

Protecting data at rest and stopping it from leaking

TPM — Trusted Platform Module

Chip on the motherboard: hardware root of trust, stores keys (e.g., BitLocker's), supports measured boot. Burned-in endorsement key uniquely identifies it. Scope: one device.

HSM — Hardware Security Module

Dedicated removable/network appliance for generating, storing, and using crypto keys at scale — tamper-resistant, used by CAs and banks. (MicroSD HSM = portable form factor.) Scope: enterprise/server.

Disk encryption

FDE — full disk encryption in software (BitLocker, FileVault); keys often TPM-protected.

SED — self-encrypting drive: encryption done in the drive hardware itself, transparent and fast. Database/file-level encryption protects specific data even from OS-level access.

Data in use

Data being processed in RAM/CPU. Protected by a secure enclave / TEE (Trusted Execution Environment) — isolated processor region even the OS can't read. Regulated data like PHI (health information, HIPAA) demands protection in all three states.

DLP — Data Loss Prevention

  • Network-based DLP — inspects outbound traffic (email, uploads) at the boundary and blocks sensitive content (SSNs, card numbers) from leaving.
  • Endpoint DLP — agent on the host controls copy/paste, printing, and removable media (block USB mass storage or allow only encrypted drives).
  • Identification methods: pattern matching, keywords, file fingerprints, classification labels.

Ownership models + the MDM control list

MDM (Mobile Device Management; UEM extends it to laptops/desktops/IoT) enforces: screen lock & encryption, remote wipe for lost/stolen devices, geofencing (policy by location — e.g., camera disabled inside the facility) vs GPS tagging (location embedded in file/photo metadata — an OPSEC leak), storage segmentation/containerization (work container separate and wipeable without touching personal data), app management via allow lists, and context-aware authentication (requirements adapt to location/network/time).

Embedded Systems, IoT & ICS

Constrained devices you can't just patch
TermDefinition
Embedded systemComputer built into a device for a dedicated function (medical devices, vehicles, printers)
IoTNetwork-connected consumer/office devices (cameras, thermostats, TVs) — notorious for default passwords and no updates → segment onto their own VLAN
ICS / SCADAIndustrial control systems for plants/utilities — availability is paramount; isolate from IT networks entirely
SoCSystem on a Chip — CPU, memory, radios on one chip (Raspberry Pi, phones)
RTOSReal-Time Operating System — guarantees deterministic timing for control systems; not patchable like a desktop OS

Constraints to cite on the exam: low compute power (can't run strong crypto or agents), inability to patch, default credentials, long deployed lifespans. Compensating controls: network segmentation, firewalls in front, monitoring.

  • EOL / legacy hardware — end-of-life means no more security patches: replace, or isolate + compensating controls. Legacy systems persist because critical software depends on them.
  • Decommissioning — before disposal: sanitize media (cryptographic erase, wipe, or physical destruction), remove from inventory, revoke its certificates/credentials. Data left on retired drives is a classic breach source.
