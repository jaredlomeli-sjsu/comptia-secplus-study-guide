Exam trick: *ARP spans Layers 2–3 — it maps L3 IP addresses to L2 MAC addresses. Mnemonic bottom-up: "Please Do Not Throw Sausage Pizza Away."

Core Protocols — TCP, UDP, IP, ICMP, ARP

The plumbing everything else rides on

TCP vs UDP Layer 4

TCPUDP
ConnectionConnection-oriented — 3-way handshake: SYN → SYN/ACK → ACKConnectionless — fire and forget
DeliveryGuaranteed, ordered, error-checkedBest-effort, no guarantee
SpeedSlower (overhead)Faster, lower latency
Used byHTTP/HTTPS, FTP, SMTP, SSH, RDPDNS queries, DHCP, TFTP, VoIP/RTP, SNMP, streaming

IP L3

Addressing + routing of packets. Connectionless — no delivery guarantee (that's TCP's job). IPv4 = 32-bit, IPv6 = 128-bit addresses.

ICMP L3

Diagnostics & error reporting — never carries app data. Tools: ping, tracert/traceroute.

abuse ICMP/ping floods (DoS) — perimeter firewalls often block ICMP.

ARP L2–3

Broadcasts "who has IP x?" → owner replies with MAC. No authentication.

attack ARP poisoning: forged replies bind attacker's MAC to a victim IP → on-path (MITM) attacks on the LAN.

Data in transit is one of the three data states (in transit / at rest / in use). Unencrypted transit traffic is readable by anyone with a sniffer — the recurring theme of this chapter is replacing cleartext protocols with encrypted ones.

Infrastructure Protocols — DNS, DHCP, NTP

Quiet services everything depends on

DNS UDP 53 TCP 53

Resolves names → IPs. UDP for queries, TCP for zone transfers (and large responses).

attacks DNS poisoning/spoofing of cached records.

fix DNSSEC — digitally signs records (authenticity/integrity; it does not encrypt).

DHCP UDP 67/68

Auto-assigns IP, subnet mask, gateway, DNS. DORA: Discover → Offer → Request → Acknowledge.

attack rogue DHCP server hands out malicious gateway/DNS.

fix DHCP snooping on switches.

NTP UDP 123

Synchronizes clocks. Critical for security: Kerberos breaks beyond ~5 min skew, and log correlation in a SIEM is meaningless without a common timeline.

IP Addressing

RFC 1918 ranges are pure memorization points

RFC 1918 — Private (non-routable) IPv4 ranges

RangeCIDRClass
10.0.0.0 – 10.255.255.25510.0.0.0/8A
172.16.0.0 – 172.31.255.255172.16.0.0/12B
192.168.0.0 – 192.168.255.255192.168.0.0/16C

Private addresses never route on the public internet — they require NAT to get out. Loopback: 127.0.0.1 (IPv4), ::1 (IPv6). APIPA self-assigned: 169.254.x.x = "I couldn't reach DHCP."

IPv4 vs IPv6

IPv4: 32-bit dotted decimal (192.168.1.5) — exhausted, hence NAT.

IPv6: 128-bit hex (2001:0db8::1) — vast space, IPsec support built into the design, no NAT needed.

Delivery methods

Unicast — one-to-one (normal traffic).

Broadcast — one-to-all on the local segment (IPv4 only; ARP/DHCP discovery use it). IPv6 replaces broadcast with multicast.

SPFThe sending server is authorized for the domainDNS TXT record listing permitted sender IPs; receiver checks the connecting IP against it

Memory hook: SPF = who may send, DKIM = signature on the message, DMARC = policy + reports tying both together. DMARC depends on SPF/DKIM — it never works alone.

Web Traffic — HTTP, TLS & IPsec

How traffic gets encrypted, at two different layers

HTTP 80 & SSL both deprecated for secure use

HTTP sends everything cleartext — credentials and session cookies are sniffable.

SSL (all versions ≤ 3.0) is broken and deprecated. If an exam answer says "use SSL," it's the wrong answer — the term survives only in marketing.

HTTPS = HTTP over TLS 443

TLS (current: 1.2 / 1.3) provides encryption, integrity, and server authentication via certificates issued by a Certificate Authority (CA) the browser trusts.

TLS 1.3 requires forward secrecy (ephemeral keys) and removed legacy ciphers.

IPsec — encryption at Layer 3

Secures any IP traffic (not just web). Two components, two modes — know all four:

PieceProvidesDetail
AH — Authentication HeaderAuthentication + integrity, no encryptionIP protocol number 51 (not a port)
ESP — Encapsulating Security PayloadEncryption + authentication + integrityIP protocol number 50 — the one actually used for confidentiality
IKE — Internet Key ExchangeNegotiates keys / security associationsUDP 500 (+ UDP 4500 NAT-traversal)
Tunnel vs Transport modeTunnel encrypts entire packet incl. header (site-to-site VPNs); Transport encrypts payload only (host-to-host inside a network)Detailed with VPNs in Ch 4
Precision point: 50 and 51 are IP protocol numbers, not ports — a favorite trick distinction. TLS encrypts a specific application session; IPsec encrypts at the network layer regardless of application.

Four variants — don't mix up SFTP and FTPS

SFTP vs FTPS: SFTP = SSH (port 22, no certificates). FTPS = TLS (certificates, FTP ports). The exam will test this pair.

Remote Access — SSH, RDP, VPN, Jump Server

Administering things you can't touch

SSH TCP 22

Encrypted command-line access; replaces Telnet 23. Also tunnels SFTP and SCP. OpenSSH is the standard implementation — supports key-based auth (ssh-keygen, public key in authorized_keys), stronger than passwords.

RDP TCP 3389

Full Windows desktop remotely. Constant brute-force/credential-stuffing target when internet-exposed.

Hardening: never expose 3389 directly — require VPN or a jump server, enable Network Level Authentication, enforce MFA and lockout.

VPN

Encrypted tunnel over an untrusted network so remote traffic is unreadable in transit. Built on IPsec or TLS. Full taxonomy (split/full tunnel, site-to-site, always-on) in Chapter 4.

Jump Server (Jump Box)

A single hardened, heavily-logged host that admins connect to first, then hop into a protected segment (DMZ, server VLAN). Centralizes the choke point: one place for MFA, session recording, and auditing of all administrative access.

VoIP & Directory Services

Two smaller protocol families

VoIP — voice over the network

SIP 5060 / 5061 TLS — sets up and tears down the call (signaling, Layer 5).

RTP — carries the actual audio/video stream over UDP.

SRTP secure — encrypted RTP; prevents eavesdropping on calls.

Memory hook: SIP dials, RTP talks, SRTP whispers.

Directory services

Central database of users, computers, and groups — Microsoft Active Directory is the canonical example.

LDAP 389 cleartext — query/modify the directory.

LDAPS 636 LDAP over TLS — required whenever directory traffic crosses untrusted segments. (LDAP injection = manipulating directory queries via unsanitized input.)

Switches Layer 2

Forwarding by MAC — and the controls that harden it

A switch learns which MAC address lives on which port and forwards frames only where they need to go (unlike a hub, which repeats everything to everyone — which is why hubs enable easy sniffing).

FeatureWhat It DoesSecurity Angle
VLANLogically segments one physical switch into isolated broadcast domains (Sales / Engineering / VoIP)Segmentation limits lateral movement; traffic between VLANs must pass a router/L3 switch where ACLs apply
Port securityLimits which/how many MAC addresses may use a switch port; violation → shut the portStops rogue devices plugged into wall jacks
MAC filteringAllow/deny list of MAC addressesweak alone — MACs are trivially spoofed; defense-in-depth only
STP / RSTPSpanning Tree Protocol detects and blocks switching loops (which cause broadcast storms)Loop prevention = availability protection; RSTP converges faster
BPDU GuardDisables a port if STP control frames (BPDUs) arrive on an access port where none belongBlocks an attacker plugging in a rogue switch to manipulate the topology

Routers, ACLs & SNMP Layer 3

Routing between networks, filtering as it goes
  • Router — connects networks and routes packets by IP; each interface is a separate broadcast domain.
  • ACL (Access Control List) — ordered allow/deny rules evaluated top-down against source/destination IP, port, and protocol. First match wins.
  • Implicit deny — anything not explicitly permitted is silently dropped by the invisible rule at the end of every ACL/firewall ruleset. The bedrock principle of rule-based filtering.

Managing network devices — SNMP

SNMP v1 / v2 UDP 161 agents · UDP 162 traps

Monitors and configures devices using "community strings" (public/private defaults) sent in cleartext. do not use

SNMPv3 use this

Same ports, adds authentication, encryption, and message integrity. The only acceptable version on the exam and in production.

Failure modes

Fail-open

Device fails → traffic flows uninspected. Chooses availability over security.

Fail-closed

Device fails → traffic is blocked. Chooses security over availability. High-security environments fail closed.

Scenario cue: "attacks against the company web application (SQLi/XSS)" → WAF, not NGFW. "One appliance for a small office combining firewall, AV, and filtering" → UTM.

Network Zones & Attack Surface

Where things live determines how exposed they are
Zone / TermDefinition
Screened subnet (DMZ)Buffer zone between the internet and the internal LAN, typically between two firewalls. Hosts public-facing services — web, mail gateway, VPN concentrator — so a compromise there doesn't directly expose the LAN. SY0-701 prefers the term screened subnet.
IntranetInternal private network — employees only.
ExtranetA controlled slice of your network shared with trusted third parties (partners, suppliers) — more open than intranet, more restricted than public.
East-west trafficServer-to-server traffic inside the data center (vs. north-south = in/out). Once attackers land, they move east-west — segmentation and Zero Trust exist to stop exactly this.
Attack surfaceThe sum of all points where an attacker could try to get in: open ports, running services, exposed APIs, user accounts. Hardening = shrinking it (disable services, close ports, remove accounts).

NAT & PAT

Private inside, public outside

NAT translates private RFC 1918 addresses to public ones at the network edge. Side benefit: internal addressing is hidden from the internet (but NAT is not a firewall).

VariantMappingUse Case

PAT (NAT overload)Many private → one public, distinguished by source port numbersWhat your home router does — the dominant form
NAT gatewayManaged NAT device/service (common cloud term)Lets private subnets reach out without being reachable inbound

SCADA & Physical Isolation

When the answer is "don't connect it at all"

SCADA / ICS

Supervisory Control and Data Acquisition — systems controlling industrial processes: power plants, water treatment, manufacturing lines.

Often runs legacy, unpatchable software → must be isolated in its own network/VLAN, never internet-reachable.

Air gap

Physical isolation — no network connection whatsoever to other networks. The strongest segmentation possible.

Remaining attack vector: removable media (Stuxnet jumped an air gap via USB) → strict USB/media policies still required.

Proxy Servers

Direction is everything: who is being protected?

Forward proxy — protects/controls clients

Sits between internal users and the internet. Provides URL/content filtering, caching, and logging of user browsing. The server sees the proxy's IP, not the user's.

Reverse proxy — protects servers

Sits in front of web servers; clients connect to it, never directly to the server. Provides TLS termination, load balancing, and hiding of the backend. WAFs commonly deploy as reverse proxies.

Exam cue: "filter which websites employees visit" = forward proxy. "Distribute requests across web servers / shield the web farm" = reverse proxy.

Zero Trust Architecture

New in SY0-701 — learn the component names exactly

Legacy: implicit trust

"Hard shell, soft center" — once you're past the perimeter (on the LAN or VPN), you're trusted everywhere. One phished laptop → free east-west movement.

Zero Trust: never trust, always verify

No location-based trust. Every request is authenticated, authorized, and encrypted — per session, regardless of where it originates.

The two planes & their components

ComponentPlaneRole
Subject—The user/device/service requesting access
Policy Engine (PE)Control planeMakes the access decision — evaluates identity, device health, behavior, threat intel against policy
Policy Administrator (PA)Control planeCommunicates the decision — tells the PEP to open or close the session (PE + PA together = the Policy Decision Point)
Policy Enforcement Point (PEP)Data planeEnforces it — the gate that actually allows/blocks the traffic between subject and resource
Enterprise resourceData planeThe app/data being protected
  • Control plane = where decisions are made (PE + PA). Data plane = where traffic flows and gets enforced (PEP ↔ resource).
  • Adaptive identity — authentication requirements adjust to risk in real time: new device + odd hour + unusual country → step-up MFA or deny.
  • ZTNA — Zero Trust Network Access: per-application access based on identity + posture, replacing broad-network VPN access. Apps stay invisible to unauthorized users.
  • SASE — Secure Access Service Edge: cloud-delivered bundle of SD-WAN + ZTNA + SWG + CASB + firewall-as-a-service, securing users wherever they are instead of backhauling to HQ.
Memory hook: PE decides, PA relays, PEP enforces. Decisions in the control plane, enforcement in the data plane. If the question says "component that allows or blocks the actual connection," it's the PEP.
