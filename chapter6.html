Attacker Attributes

The three axes the exam classifies actors by

Internal vs External

Internal — insiders with legitimate access (employees, contractors). Bypass perimeter defenses entirely.

External — everyone outside; must breach a boundary first.

Resources / Funding

Ranges from a free laptop (unskilled) to government budgets (nation-state). Funding determines tooling, persistence, and whether they can buy zero-days or insiders.

Level of Sophistication

From running scripts blindly to developing custom malware. High sophistication = stealth, anti-forensics, living-off-the-land techniques.

Exam cue: the motivation usually names the actor. Money → organized crime. Beliefs → hacktivist. Revenge → insider. Espionage/war → nation-state. "With written permission" → ethical hacker, no matter what they did.

The paths an attack travels to reach you

Supply chain is the trick answer: if the scenario says the attacker got in "through the company that manages our IT" or via a tainted software update, that's the supply chain vector via an MSP — not phishing, not an insider.

Attack Surface & Shadow IT

What's exposed — including what you don't know about

Attack surface

Every point an attacker could touch: open ports, running services, exposed APIs, user accounts, physical access, and people. Every new system, vendor, or app grows it; hardening shrinks it.

Shadow IT

Systems, apps, and cloud services employees use without IT approval — a personal Dropbox for work files, an unsanctioned SaaS tool. Invisible to security: unpatched, unmonitored, unbacked-up. Detected with CASB and network monitoring.

Malware Types

One-line identifiers — the exam tests the distinctions
TypeBehaviorIdentify It By
VirusAttaches to a host file/program; replicates when the user runs itNeeds user action to spread
WormSelf-replicates over the network — no user action, no host file"Spread to every machine overnight by itself"
Trojan horseMalware disguised as legitimate software (cracked games, fake utilities)User installed it on purpose, thinking it was something else
RAT — Remote Access TrojanTrojan that gives the attacker remote control — a backdoor"Attacker controlled the machine remotely"
Logic bombDormant code that triggers on an event or date"Executed when the admin was removed from payroll" — classic insider revenge
KeyloggerRecords keystrokes (software or hardware dongle) to steal credentialsCaptured passwords despite encryption in transit
SpywareSecretly monitors activity and collects data; privacy-invasive software does this semi-openly via buried consentBrowser redirects, harvested browsing data
RootkitEmbeds at kernel/system level to hide itself and other malware"AV finds nothing but the system acts compromised" — needs boot-time/offline scan
RansomwareEncrypts your data, demands payment; modern crews also exfiltrate first (double extortion)Files inaccessible + ransom note; defense = offline backups
BloatwarePreinstalled unwanted software — not malicious, but unpatched attack surfaceRemove during hardening / gold-image builds
Virus vs worm is the most-tested pair: virus needs a user to run the infected file; a worm spreads itself across the network. Trojan vs RAT: every RAT is a trojan, but the RAT's defining feature is remote control.

Data exfiltrationLarge/unusual outbound transfers, especially off-hours or to unfamiliar destinations
Unexpected encrypted trafficMalware hides its command channel in encryption a host doesn't normally use

These pair with the host-side anomalies from Chapter 2 (resource spikes, inaccessible files, log gaps) — the network view and host view of the same compromise.

Blocking Malware

Layered defenses — and how detection actually works

Spam filter

At the email gateway — strips the #1 delivery vehicle before users ever see it. Pairs with SPF/DKIM/DMARC from Chapter 3.

Anti-malware

On every host and at the gateway. Detection methods below — signature for the known, heuristic for the new.

UTM

All-in-one network appliance: firewall + IPS + anti-malware + content filtering in one box at the boundary.

Signature-based detection

Matches files against a database of known malware fingerprints. Accurate, low false positives — but blind to brand-new malware until signatures update. Why definitions must auto-update.

Heuristic / behavior-based detection

Runs suspicious code (often in a sandbox) and flags malicious behavior — even with no signature. Catches zero-day malware; pays for it with more false positives.

File Integrity Monitoring (FIM) — baselines hashes of critical system files and alerts on any change. Catches what scanners miss: if a rootkit alters a system binary, the hash no longer matches. (Tripwire is the classic example.)

Threat Intelligence Sources

Where defenders learn what attackers are doing
SourceWhat It Is
OSINT — Open Source IntelligenceAnything publicly available: websites, news, social media, DNS records, public breach data. Free — and attackers use it for recon too.
Closed / proprietary intelligencePaid commercial feeds and private research — a vendor's trade secret / intellectual property, which is why it costs money.
Vulnerability databasesCVE (MITRE's catalog of known vulnerabilities) and NVD (NIST's database adding CVSS severity scores).
STIX — Structured Threat Information eXpressionThe standardized language/format for describing threat data (IoCs, TTPs, actors).
TAXII — Trusted Automated eXchange of Intelligence InformationThe transport protocol that moves STIX data between organizations automatically.
AIS — Automated Indicator SharingCISA's program using STIX/TAXII to share indicators between the US government and private sector in near-real-time.
Dark webMonitoring criminal markets/forums for your stolen credentials, data, or planned attacks.
Information-sharing organizationsPublic/private groups (ISACs/ISAOs) where industry peers share threat data — e.g., FS-ISAC for financial services.
IoC — Indicators of CompromiseThe atoms of threat intel: malicious IPs, file hashes, domains, registry keys that signal a known attack.
Predictive analysisUsing current intel + trends to anticipate future attacks rather than react.
Threat mapsReal-time visualizations of global attack traffic — broad awareness more than actionable detail.
File / code repositoriesVirusTotal (check files/hashes against dozens of engines), GitHub (where leaked exploits — and your accidentally-committed secrets — live).
